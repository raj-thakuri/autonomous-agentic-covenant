# Security Policy

## Reporting a Vulnerability

If you discover a security issue in this repository — including 
sensitive information inadvertently committed, a vulnerability in 
any tooling or scripts, or content that could cause harm if 
exploited — please report it privately rather than opening a 
public issue.

Use GitHub's private vulnerability reporting:
Repository → Security → Report a vulnerability

We aim to respond within 5 business days.

## Scope

This repository contains governance framework documents and 
supporting artifacts. Security reports are most relevant for:

- Sensitive information accidentally committed to the repository
- Vulnerabilities in any scripts or tooling contributed to /tools
- Content errors that could cause harm if followed as written

Proposed changes to framework principles are not security 
vulnerabilities — please use the standard Issues process 
described in CONTRIBUTING.md.
